Read View 구조
RV
Read View
SELECT 최초 실행 시 생성되는 스냅샷 정보
[]
m_ids (활성 트랜잭션 목록)
Read View 생성 시점에 아직 커밋되지 않은 트랜잭션 ID 목록입니다. 이 목록에 있는 트랜잭션이 수정한 데이터는 볼 수 없습니다.
[101, 105, 107]
min
m_up_limit_id (하한선)
m_ids 중 가장 작은 값입니다. 이 값보다 작은 트랜잭션이 수정한 데이터는 무조건 볼 수 있습니다.
101 (m_ids 중 최소값)
max
m_low_limit_id (상한선)
Read View 생성 시점에 아직 할당되지 않은 트랜잭션 ID입니다. 이 값 이상의 트랜잭션이 수정한 데이터는 무조건 볼 수 없습니다.
110 (다음에 할당될 ID)
ID
m_creator_trx_id
Read View를 생성한 트랜잭션의 ID입니다. 자기 자신이 수정한 데이터는 항상 볼 수 있습니다.
108 (현재 트랜잭션)
가시성 판단 로직
1
DB_TRX_ID == m_creator_trx_id
보임
내가 수정한 데이터는 항상 볼 수 있습니다.
2
DB_TRX_ID < m_up_limit_id
보임
하한선보다 작으면 Read View 생성 전에 이미 커밋된 트랜잭션입니다.
3
DB_TRX_ID >= m_low_limit_id
안 보임
상한선 이상이면 Read View 생성 이후에 시작된 트랜잭션입니다.
4
DB_TRX_ID in m_ids
확인 필요
m_ids에 있으면 아직 커밋 안 됨 → 안 보임
m_ids에 없으면 이미 커밋됨 → 보임
예시
트랜잭션 108이 SELECT 실행
m_creator_trx_id
108
m_ids (미커밋 트랜잭션)
[101, 105, 107]
m_up_limit_id (하한선)
101
m_low_limit_id (상한선)
110
레코드의 DB_TRX_ID가 103이라면?
1. 108(내 ID)과 같은가? → 아니오
2. 101(하한선)보다 작은가? → 아니오 (103 > 101)
3. 110(상한선) 이상인가? → 아니오 (103 < 110)
4. m_ids [101, 105, 107]에 있는가? → 아니오

→ 결론: 103은 m_ids에 없으므로 이미 커밋됨 → 보임
격리 수준에 따른 Read View 생성 시점
REPEATABLE READ: 트랜잭션의 첫 SELECT 때 Read View를 한 번만 생성하고 계속 사용합니다.
READ COMMITTED: 매 SELECT마다 새로운 Read View를 생성합니다.
