버전 체인 길이 증가

트랜잭션 타임라인
T0
Tx A (trx_id=100)
BEGIN - 트랜잭션 시작
T1
Tx B (trx_id=101)
UPDATE val=1 → val=2, COMMIT
T2
Tx C (trx_id=102)
UPDATE val=2 → val=3, COMMIT
T3
Tx D (trx_id=103)
UPDATE val=3 → val=4, COMMIT
T4
Tx E (trx_id=104)
UPDATE val=4 → val=5, COMMIT
Tx A 여전히 열려있음 (COMMIT 안 함)
버전 체인 상태
Buffer Pool (현재 레코드)
val = 5, trx_id = 104
↓
val = 4 trx_id = 103
↓
val = 3 trx_id = 102
↓
val = 2 trx_id = 101
↓
val = 1 trx_id = 99 (Tx A가 볼 버전)
X Purge Thread: Tx A가 val=1을 봐야 하므로 중간 버전 삭제 불가
문제점
Tx A가 나중에 SELECT하면 val=1(trx_id < 100)을 봐야 합니다.
체인을 따라 val=5 → val=4 → val=3 → val=2 → val=1 순서로 탐색해야 합니다.
체인이 길수록 탐색 시간이 증가하여 SELECT 성능이 저하됩니다.
