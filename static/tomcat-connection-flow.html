핵심 포인트

TCP 3-way handshake가 완료된 연결이 acceptCount 큐에서 대기합니다. Acceptor가 accept()하면 Poller에 등록되고, maxConnections까지 연결을 유지합니다. 이 단계에서는 아직 HTTP 요청을 처리하지 않습니다.

일반 ThreadPoolExecutor

  • 1corePoolSize까지 스레드 생성
  • 2core 다 차면 큐에 먼저 넣음
  • 3큐 다 차면 maxPoolSize까지 스레드 추가
  • 4max도 다 차면 reject

Tomcat TaskQueue

  • 1corePoolSize까지 스레드 생성
  • 2core 다 차면 maxPoolSize까지 스레드 먼저 생성
  • 3max도 다 차면 그제서야 큐에 넣음
  • 4큐도 다 차면 reject

Tomcat이 다른 이유

일반 ThreadPoolExecutor는 큐를 먼저 채우고 스레드를 나중에 만듭니다. 하지만 Tomcat은 스레드를 먼저 최대한 만들고 큐는 마지막에 씁니다. 이렇게 하면 요청이 큐에서 대기하는 시간이 줄어들어 응답 지연이 감소합니다.
