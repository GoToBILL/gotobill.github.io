NIO vs NIO2 요청 처리 흐름

NIO (Selector 기반)
1
클라이언트 요청 도착
데이터가 NIC → 커널 수신 버퍼에 도착
2
Poller가 이벤트 감지 epoll
Selector.select()가 "읽기 가능" 이벤트 반환
selector.select(); // 이벤트 감지만
3
Worker 스레드에 작업 할당
Poller가 ThreadPool에 작업 전달
executor.execute(socketProcessor);
4
Worker가 직접 read() 블로킹
커널 버퍼 → 유저 버퍼 복사
복사 완료까지 Worker 스레드 대기
channel.read(buffer); // 여기서 복사
비즈니스 로직
5
Spring 코드 실행
HTTP 파싱 → Controller → Service → Repository
응답 전송
6
Worker가 직접 write() 블로킹
유저 버퍼 → 커널 송신 버퍼 복사
channel.write(response); // 여기서 복사
7
OS가 네트워크 전송 커널
커널 송신 버퍼 → NIC → 클라이언트
NIO2 (Async I/O)
1
클라이언트 요청 도착
데이터가 NIC → 커널 수신 버퍼에 도착
2
OS가 read 완료 커널이 처리
커널 버퍼 → 유저 버퍼 복사까지 OS가 수행
// OS가 백그라운드에서 완료
3
콜백 호출 논블로킹
I/O 완료 후 Worker 스레드에서 콜백 실행
buffer에 이미 데이터 있음
handler.completed(result, attachment);
비즈니스 로직
4
Spring 코드 실행
HTTP 파싱 → Controller → Service → Repository
응답 전송
5
write 요청 + 콜백 등록 논블로킹
요청만 하고 즉시 리턴
channel.write(buf, handler); // 바로 리턴
6
OS가 write + 네트워크 전송 커널
유저 버퍼 → 커널 버퍼 → NIC → 클라이언트
완료 시 콜백 호출
핵심 차이점
구분 NIO NIO2
이벤트 감지 Poller (Selector.select) 없음 (OS가 완료 통지)
read/write 수행 Worker 스레드가 직접 OS 커널이 수행
I/O 중 스레드 블로킹 대기 다른 일 가능
비유 "준비되면 알려줘, 내가 가져갈게" "다 되면 가져다줘"
