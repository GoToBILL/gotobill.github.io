InnoDB MVCC 히든 컬럼 구조
InnoDB 레코드 내부 구조
트랜잭션 ID 6 bytes DB_TRX_ID
롤백 포인터 7 bytes DB_ROLL_PTR
Row ID 6 bytes DB_ROW_ID
사용자 데이터 가변 col1, col2, ...
히든 컬럼 상세
T
DB_TRX_ID
트랜잭션 식별자
해당 레코드를 마지막으로 수정한 트랜잭션의 ID를 저장합니다. MVCC에서 데이터 가시성을 판단하는 핵심 컬럼입니다.
크기 6 bytes
최대값 281조
용도 스냅샷 읽기
R
DB_ROLL_PTR
언두 로그 포인터
언두 로그 세그먼트에 저장된 이전 버전 레코드를 가리키는 포인터입니다. 롤백과 MVCC 읽기에 사용됩니다.
크기 7 bytes
구성 rseg + page + offset
용도 버전 체인 탐색
I
DB_ROW_ID
묵시적 Row ID
PK나 Unique NOT NULL 인덱스가 없는 테이블에서 InnoDB가 자동 생성하는 클러스터링 키입니다.
크기 6 bytes
생성 조건 PK 없을 때만
타입 단조 증가
DB_ROLL_PTR을 통한 버전 체인
현재 레코드
DB_TRX_ID 150
name '김철수'
age 30
→
Undo Log #1
DB_TRX_ID 120
name '김철수'
age 28
→
Undo Log #2
DB_TRX_ID 100
name '김영희'
age 25
→
...
더 오래된 버전들
트랜잭션 130에서 시작된 SELECT는 DB_TRX_ID가 130 이하인 버전을 찾아 Undo Log #1의 데이터(age=28)를 읽습니다.
이것이 MVCC의 핵심 원리입니다. 각 트랜잭션은 자신의 스냅샷 시점에 맞는 데이터를 읽을 수 있습니다.
DB_ROW_ID 생성 조건
PK가 있는 테이블
사용자가 정의한 PK가 클러스터링 키로 사용됩니다.
DB_ROW_ID 컬럼이 생성되지 않습니다.
CREATE TABLE users (
  id BIGINT PRIMARY KEY,
  name VARCHAR(100)
);
PK가 없는 테이블
InnoDB가 DB_ROW_ID를 자동 생성하여 클러스터링 키로 사용합니다.
6바이트 크기로 단조 증가합니다.
CREATE TABLE logs (
  message TEXT,
  created_at DATETIME
);
요약
DB_TRX_ID - 데이터 가시성 판단
DB_ROLL_PTR - 이전 버전 탐색
DB_ROW_ID - 묵시적 PK
