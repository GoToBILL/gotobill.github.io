윈도우 너무 작음

  • • 요청을 순차적으로 전송
  • • 응답 시간 증가
  • • 처리량 감소

최적 윈도우 크기

  • • 병렬성과 혼잡 제어 균형
  • • 밀리초 이하 응답 시간
  • • 네트워크 효율 최대화

윈도우 너무 큼

  • • Incast Congestion 발생
  • • 패킷 손실 증가
  • • 스위치 과부하

핵심 포인트

  • • 슬라이딩 윈도우는 모든 Memcache 요청에 독립적으로 적용 (TCP 윈도우는 단일 스트림에만 적용)
  • • 동적으로 윈도우 크기를 조절하여 네트워크 상태에 적응
  • • 스위치와 네트워크 장비의 과부하를 방지
